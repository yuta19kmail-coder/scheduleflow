# MHS 本番化セットアップ手順（mhs.kobayashi-motors.com）

> 🎯 v0.6.0 でアプリ側のコードは本番接続済み。あとは **Firebase コンソール・DNS・OAuth・デプロイ** の設定だけ。
> CarFlow / StockFlow / CoreFlow と同じ Firebase プロジェクト **carflow-9d500** に相乗りします。
> ⚠ 順番が大事。**A→B→C→D が終わるまで、ログインは通りません**（authDomain がアプリと同じサブドメインのため）。

---

## A. Hosting にサイトを作る（Firebase コンソール）
1. Firebase コンソール → プロジェクト **carflow-9d500** → **Hosting**
2. 「**別のサイトを追加**」→ サイトID = **`mhs-km`** で作成

## B. 一旦デプロイして表示確認（PowerShell・`D:\アプリ開発\MHS` で）
```powershell
cd D:\アプリ開発\MHS
firebase target:apply hosting mhs mhs-km
firebase deploy --only hosting:mhs
```
→ `https://mhs-km.web.app` が開けば配信OK（この時点ではログインはまだ通らなくて正常）。

## C. 独自ドメイン mhs.kobayashi-motors.com を割り当て
1. Hosting → **mhs-km** → 「**カスタムドメインを追加**」→ `mhs.kobayashi-motors.com`
2. 表示される **DNS レコード**（A 2件 など）を、ドメイン管理（お名前.com 等）に追加
3. 反映（数十分〜数時間）＋ SSL 証明書の発行を待つ

## D. ログインを通すための承認設定（これが無いとログイン不可）
1. Firebase コンソール → **Authentication → Settings → 承認済みドメイン** に
   `mhs.kobayashi-motors.com` を**追加**
2. **Google Cloud Console** → 同プロジェクト → APIとサービス → 認証情報 →
   OAuth 2.0 クライアント（ウェブ）→ **承認済みのリダイレクト URI** に
   `https://mhs.kobayashi-motors.com/__/auth/handler` を**追加**
   （StockFlow の `…stockflow.kobayashi-motors.com/__/auth/handler` と同じ要領）

## E. Firestore ルールを更新（社内予定コレクションを許可）
- `D:\アプリ開発\CarFlow\carflow\firestore.rules` に **MHS の scheduleEvents ブロックを追加済み**（既存は一切変更なし・追加だけ）。
- 反映方法（どちらか）：
  - Firebase コンソール → **Firestore → ルール** に firestore.rules の**全文を貼って「公開」**、または
  - CarFlow 側で `firebase deploy --only firestore:rules`
- 会社カレンダーのマスター（休業ルール/行事）は `settings/mhsCompanyCalendar` に保存され、**既存の settings ルール**で保護されるので追加設定は不要。

## F. 動作確認
1. `https://mhs.kobayashi-motors.com` を開く → **Google ログイン**
2. CoreFlow 名簿（portalMembers）に居るアカウントなら入室。名簿に無いと「🔒 権限がありません」。
3. **＋社内予定**で予定を追加 → **別の端末でも同じ予定が見える**ことを確認（Firestore 同期）。
4. **⚙️会社カレンダー設定**で休業ルールを足す → カレンダーに反映されることを確認。
